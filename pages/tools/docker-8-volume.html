# 数据卷详解

# 数据卷作用

完成容器与宿主机之间的==资源共享==

# 数据卷特点

  1. 数据卷可以在容器之间共享和重用
  2. 对数据卷的修改会立即影响到对应的容器
  3. 对数据卷的更新不会影响镜像
  4. 数据卷一直存在，即使容器被删除

注意：数据卷 的使用，类似于 Linux 下对目录或文件进行 mount，镜像中的被指定为挂载点的目录中的文件会复制到数据卷中（仅数据卷为空时会复制）。

# 命令

# 创建数据卷(自动、手动)
docker run -v 宿主机绝对路径|任意别名:/容器内的路径 镜像名
docker run -v 宿主机绝对路径|任意别名:/容器内的路径:ro 镜像名	# ro容器只能读不能写
docker volume create 数据卷名

docker run -d -p 8080:8080 --name tomcat01 -v /root/apps:/usr/local/tomcat/webapps tomcat:8.0-jre8.0		# 只能在启动时指定数据卷且启动后会覆盖容器内目录下原有文件
docker run -d -p 8080:8080 --name tomcat01 -v aa:/usr/local/tomcat/webapps tomcat:8.0-jre8.0		# aa代表一个数据卷名字，随便写，不存在则自动创建数据卷同时自动映射宿主机中某个目录，同时将容器目录文件复制到宿主机中
docker volume create bb
# 查看数据卷
docker volume ls
docker volume inspect 卷名

docker volume inspect aa
# 删除数据卷
docker volume prune 		# 移除所有未使用的
docker volume rm 卷名		  # 移除指定的

docker volume rm aa
