# 队列持久化

保障当 RabbitMQ 服务停掉以后消 息生产者发送过来的消息不丢失。默认情况下 RabbitMQ 退出或由于某种原因崩溃时，它忽视队列和消息，除非告知它不要这样做。确保消息不会丢失需要做两件事：我们需要将队列和消息都标 记为持久化。

# 代码实现

声明队列方法的第二个参数，布尔类型，是否持久化

//持久化
boolean durable = true;
channel.queueDeclare(QUEUE_NAME, durable, false, false, null);

# 报错

Caused by: com.rabbitmq.client.ShutdownSignalException: channel error; protocol method: #method<channel.close>(reply-code=406, reply-text=PRECONDITION_FAILED - inequivalent arg 'durable' for queue 'ack_queue' in vhost '/': received 'true' but current is 'false', class-id=50, method-id=10)

原因：该队列已经存在了并且是不持久化的，此时又声明这个队列持久化，造成冲突

解决：将该队列删除，再重新声明的时候就不会报错了

启动成功后看到大写 D 标识，Durable标记

# 消息持久化

队列持久化以后消息不持久化，消费消息失败的时候，消息仍会丢失

# 代码实现

生产者发布消息的时候，修改消息的属性为文本持久化

//设置生产者发送的消息为持久化消息（保存在磁盘上，否则保存在内存中）
channel.basicPublish("", QUEUE_NAME, MessageProperties.PERSISTENT_TEXT_PLAIN, message.getBytes(StandardCharsets.UTF_8));

# 安全性

非绝对安全，即使开启了两种持久化，消息仍有可能丢失（刚准备保存到磁盘的时候，还没存储完时出错）
